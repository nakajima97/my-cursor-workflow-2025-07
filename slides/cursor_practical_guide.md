## Cursorとは？

### AI搭載のコードエディタ
- **VSCodeベース** のインターフェース
- **最新AIモデル**（Claude 4、GPT-4等）を内蔵
- **リアルタイムでコード生成・修正**をサポート

### 従来のエディタとの違い
✅ AIがコード提案・生成を支援  
✅ 自然言語でプログラミング可能  
✅ バグ修正や機能追加を対話で実現  

---

## なぜCursorを選ぶのか？

### 他のAIツールとの比較優位性

**馴染みやすさ:**
- VSCodeそっくりのUI
- VSCodeのショートカットキーでそのまま使えるものが多い
- 学習コストが低い

**利用者が多い**
- ネットに情報が多い
- 大手企業への導入事例が多い

## 基本機能を覚えよう

---

## 1. Chat機能（Ctrl+L）

### 用途
- **質問・相談** - コードの説明、エラー解決
- **簡単なコード生成** - 関数やクラスの作成
- **リファクタリング提案** - コード改善案

### 使い方のコツ
💡 **具体的に質問する**  
❌ 「このコードどう？」  
✅ 「この関数のパフォーマンスを改善したい」

---

## 2. Composer機能（Ctrl+Shift+I）

### 用途
- **ファイル横断的な作業** - 複数ファイルの同時編集
- **機能の実装** - まとまった機能の追加
- **プロジェクト構造の変更** - ディレクトリ作成等

### Chat vs Composer
| Chat | Composer |
|------|----------|
| 単一ファイル中心 | 複数ファイル対応 |
| 質問・相談向け | 実装作業向け |
| 軽い作業 | 重い作業 |

---

## 3. Diff機能

### 概要
- AIが提案した**変更内容を事前確認**
- **Accept/Reject**を選択可能
- **部分的な採用**も可能

### 重要性
🚨 **必ず内容を確認してからAccept**  
🔍 **理解できない変更は一旦Reject**  
📝 **疑問があればChatで質問**

## ショートカット設定

### 重複削除が必要なキー
- `Ctrl + L` （Chat起動）
- `Ctrl + Shift + K` （ターミナル機能等）
- `Ctrl + K` （コマンドパレット等）

### 設定方法
1. **File > Preferences > Keyboard Shortcuts**
2. 該当するショートカットを検索
3. **Delete**キーで削除
4. **代替キーは設定しない**（シンプルに保つ）

---

## 料金プラン選択

### 推奨設定：Pro版
- **月額$20**
- **月500回制限**で十分
- **無制限プラン回避** - 制限リスクを避ける

### 設定手順
1. **Settings > Subscription**
2. **Pro Plan**を選択
3. **Opt Out of New Pricing**にチェック

**重要:** 初心者は月500回で十分です

## モデル選択

### 推奨設定：Claude-4-Sonnet固定

### 設定理由
✅ **一貫した回答品質**  
✅ **Autoモードの予期しない動作を回避**  
✅ **最新モデルの安定利用**

### 設定方法
**Settings > Models > Default Model**
→ **Claude-4-Sonnet**を選択

## プロンプト戦略（5ステップサイクル）

### 初心者でも使える改善サイクル

1. **Composerに素直に指示**
   - 「ログイン画面を作って」

2. **うまく行ったらOK**
   - そのまま次のタスクへ

3. **失敗したら原因調査**
   - エラーメッセージを確認

4. **ChatGPTでプロンプト改善**
   - 「この指示をもっと具体的にして」

5. **改善版をCursorに投入**
   - より詳細な指示で再実行

## User Rules活用

### ショートカットエイリアスの設定

**Settings > Rules for AI > User Rules**に以下を追加：

```
/plan - 計画の認識合わせ（重要な判断前に使用）
/ask - ポリシー相談（迷ったときに使用）
/debug - バグ原因特定（エラー発生時に使用）
/cmt - コメント・ドキュメント追加
/log - ログ追加
```

### 使用例
- **実装前:** `/plan このログイン機能の実装計画を立てて`
- **エラー時:** `/debug このエラーの原因を5つ候補を出して`

## MCPとは？

### Model Context Protocol
- **外部ツールとの連携機能**
- **リアルタイムデータアクセス**
- **作業履歴の外部保存**

### 初心者におすすめのMCP
1. **GitHub MCP** - Issue管理、履歴保存
2. **Context7** - ライブラリドキュメント参照

---

## GitHub MCP

### 用途
✅ **Issue駆動開発** - タスク管理  
✅ **プロンプトメモの外部保存** - 失敗履歴管理  
✅ **実装履歴管理** - 過去の作業参照

### なぜ必要？
💾 **履歴クリア対策** - Cursor の会話履歴は消える  
📊 **失敗分析** - 何がうまくいかなかったかを記録

---

## Context7

### 機能
- **ライブラリ・フレームワークの最新ドキュメント**
- **コード例とAPI仕様**
- **リアルタイム参照**

### 対応ライブラリ（例）
- React, Next.js, Vue
- Express, FastAPI
- MongoDB, PostgreSQL
